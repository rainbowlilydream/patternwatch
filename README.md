# 🔮 PatternWatch

**What if we could see the signals before the event?**

Before earthquakes, volcanic eruptions, and other major events, people often notice small things: a pet acting strange, an unusual dream, something feeling "off."

These observations are usually forgotten — until after something happens.

**PatternWatch collects these signals continuously**, so we can look back and find patterns that no one person could see alone.

---

## How It Works

1. **Notice something unusual?** Report it. Takes 10 seconds.
2. **We timestamp and map it.** City-level location only.
3. **Patterns emerge.** When many people report similar things in the same area.
4. **After events, we correlate.** What was logged before the earthquake? We check.

---

## What We Collect

| Category | Examples |
|----------|----------|
| 🐕 Pet/animal behavior | Restless, hiding, barking at nothing |
| 😴 Unusual dreams | Vivid, disturbing, precognitive |
| 🏠 Home/structural | Cracks, settling sounds, things falling |
| 😵 Body sensations | Headache, dizziness, ear pressure |
| 📱 Electronics | Interference, glitches, GPS drift |
| 💧 Water/well | Level changes, temperature, color |
| 🌀 Just a feeling | Unease, "something's off" |
| ✨ All good | Normal day (baseline matters!) |

## Principles

- **Open data** — Anyone can analyze. Researchers welcome.
- **No predictions** — We show patterns, not forecasts.
- **Privacy first** — City-level location only. No accounts required.
- **Transparent** — Open source. No black boxes.
- **No profit** — Public infrastructure, not a business.

---

## The Key Insight

Existing systems ask "did you feel it?" **after** earthquakes.

We ask "what did you notice?" **every day** — before anyone knows what's coming.

The difference: clean, unbiased, timestamped data.

npm run dev
